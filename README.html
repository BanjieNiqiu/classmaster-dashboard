统双班务网页项目文档

Technical Report | Version 1.0

一、项目概览与核心价值

统双班务网页（Class Affairs System）是一款专为高校班级设计的全栈Web应用，旨在解决大二下学期以来课程信息繁杂、作业与考试安排分散、资料难以统一查找等实际问题。系统通过现代化技术架构与AI能力集成，为班级构建一个集信息查询、事务管理、资源共享于一体的“第二空间”，显著提升学习协作效率。

本项目已实现完整交付，包含前端页面、后端API、配置文件及部署文档，所有代码均基于Vercel Serverless Functions与MongoDB构建，具备高可用性与可扩展性。核心价值体现在以下四个方面：

  • 信息聚合中枢：将课程情报、班务任务、物资借用、文档资源等碎片化信息集中管理，提供统一入口。
  • 四层权限体系：支持开发者、班委、普通同学、游客四种角色登录，功能菜单与操作权限精准匹配身份。
  • AI智能增强：集成LLM智能问答助手，支持自然语言查询作业安排、考试时间等，结合RAG技术确保回答准确并附带来源引用。
  • 一键部署就绪：项目结构清晰，依赖明确，配合详细的GitHub上传与Vercel配置指南，可快速上线运行。

系统采用模块化设计，前后端分离架构，便于后续功能拓展与维护，是高校学生团队实现数字化班级管理的理想解决方案。

二、系统功能详细说明

统双班务网页系统围绕班级日常事务管理的核心需求，构建了七大功能模块，覆盖信息查询、事务处理、资源共享与智能交互等全场景。各模块均实现前后端联动，并通过四层权限体系保障数据安全与操作合规。

1. 用户认证管理系统

系统支持四种角色登录：开发者（Admin）、班委（Monitor）、普通同学（Student） 和 游客（Guest）。不同角色在登录后将看到差异化的功能菜单与操作权限，确保用户仅能访问其职责范围内的资源。

  • 开发者：拥有最高权限，可访问后台管理面板，进行系统配置、监控与审计。
  • 班委：具备任务发布、物资管理、情报核实等管理能力。
  • 普通同学：可提交表单、借用物资、查询课程信息、上传文档。
  • 游客：仅可浏览公开信息，无任何写入或敏感操作权限。

所有用户身份通过JWT令牌进行验证，密码采用bcrypt加密存储，保障账户安全。

2. 班务收集管理系统

该模块为班委提供完整的任务生命周期管理能力，支持动态表单的创建与数据收集。

功能项 能力说明 权限角色
任务创建 定义任务标题、描述、截止时间，设置可见性（公开/班级内） 班委
动态表单 支持文本、数字、日期、单选、多选、文件上传等多种字段类型，可调整顺序与必填状态 班委
表单填写 同学可在线填写并提交表单，支持本地暂存与自动保存 普通同学
数据导出 将收集结果一键导出为Excel文件，便于线下统计分析 班委

3. 物资共享与借用系统

实现班级公共物资与个人闲置物品的闭环管理，包含“可借用 → 已借出 → 待确认归还”三态流转机制。

状态 触发动作 操作者 结果
可借用（绿色） 点击“借用”按钮 普通同学 状态变更为“已借出”，记录借用人与时间
已借出（红色） 点击“归还”按钮 借用人 状态变更为“待确认归还”
待确认归还（黄色） 点击“确认收回” 所有者（班委或个人） 状态恢复为“可借用”，完成归还流程

系统支持按所有者类型筛选物资，班委可查看班级资产流向明细并导出清单。

4. 课程情报与作业查询系统

整合课程相关信息，支持情报发布、点赞核实与AI智能问答，提升信息可信度与获取效率。

  • 情报分类：作业通知、考试安排、课件资料、重要通知。
  • 互动机制：用户可对情报进行点赞（upvotes），班委可标记为“verified”以增强可信度。
  • 时间敏感提示：即将截止的情报突出显示，过期内容自动归档但仍可检索。
  • LLM智能助手：集成自然语言问答功能，用户提问如“这周高数有什么作业？”系统将检索相关情报并生成摘要回答。

5. 文档资源共享系统

提供结构化文档管理平台，支持多格式文件上传、版本控制与下载统计。

功能 说明
分类管理 按课程（如高等数学、英语）和类型（课件/笔记/真题/工具）组织文档
版本控制 同名文件自动添加版本号（如_v2）区分，避免重复上传
重要标记 班委可标记关键资料，置顶推荐
下载统计 实时更新下载次数，高频文档优先展示

6. LLM智能问答助手

基于检索增强生成（RAG）技术，实现语义级知识服务。

  • 接收自然语言问题，调用LLM模型生成回答。
  • 自动检索course_intel数据库中最相关的条目作为上下文输入。
  • 回答附带原始情报链接作为引用来源，确保信息可追溯。
  • 支持对话历史保存，维持上下文连贯性。

7. 后台管理系统

专为开发者设计的运维中心，涵盖系统配置、实时监控与数据维护。

  • LLM服务配置：可视化管理API密钥、切换模型版本、设置请求超时。
  • 系统参数调节：滑块控制文件上传大小限制、任务默认有效期等运行参数。
  • 实时仪表盘：展示CPU、内存、API调用频率等核心指标。
  • 操作审计日志：完整记录管理员操作轨迹，支持搜索与CSV导出。
  • 数据维护工具：提供缓存清理、索引重建、备份恢复等高级功能。

三、技术架构与实现方案

统双班务网页采用现代化全栈架构设计，以前后端分离模式实现高内聚、低耦合的系统结构。整体技术栈围绕Vercel Serverless Functions构建，确保应用具备弹性伸缩、免运维和快速部署的优势。各核心组件均选用行业主流技术方案，兼顾开发效率与运行稳定性。

前端架构

前端基于轻量级技术组合实现响应式用户界面，无需复杂构建流程即可高效运行。

  • HTML5 + Tailwind CSS：采用实用优先的CSS框架，实现现代化UI组件与移动端适配。
  • JavaScript（原生）：通过内联脚本处理页面交互逻辑，减少外部依赖，提升加载速度。
  • CDN资源集成：引入Font Awesome图标库与ECharts可视化图表库，增强界面表现力。
  • 响应式布局：所有页面均适配桌面与移动设备，保障多端使用体验一致性。

后端架构

后端服务部署于Vercel Serverless Functions环境，以无状态函数形式提供RESTful API接口。

  • Node.js + Express：作为核心运行时与Web框架，处理HTTP请求路由与中间件逻辑。
  • 模块化API设计：功能按业务域拆分为独立接口文件，包括：
    • users.js：用户认证与权限管理
    • tasks.js：班务任务与表单收集
    • items.js：物资借用状态流转
    • courses.js：课程情报发布与检索
    • documents.js：文档上传与版本控制
    • llm.js：智能问答与语义搜索
    • admin.js：后台配置与系统监控
  • CORS支持：所有接口默认启用跨域资源共享，确保前后端通信畅通。
  • 错误处理机制：统一返回标准化JSON格式错误信息，便于前端解析与调试。

数据存储方案

系统采用分层数据管理策略，结合数据库与对象存储服务满足不同类型的数据需求。

存储类型 技术选型 应用场景
结构化数据 MongoDB 用户信息、任务定义、物资状态、课程情报等
非结构化文件 对象存储服务（OSS） 文档资源（PDF/Word/PPT等）、图片附件
缓存与会话 Vercel边缘缓存 页面静态资源、API响应缓存、JWT令牌验证

LLM集成与AI能力

系统深度集成大语言模型，通过检索增强生成（RAG）技术提升信息服务智能化水平。

  • 自然语言接口：用户可通过提问方式查询作业安排、考试时间等信息。
  • 语义检索机制：将问题转化为向量嵌入，在course_intel集合中进行相似度匹配。
  • 上下文增强生成：将最相关的情报条目作为上下文输入LLM，生成准确且可追溯的回答。
  • 引用溯源功能：AI回答自动附带原始情报链接，确保信息来源透明可信。

认证与权限体系

系统构建了四层角色权限控制模型，确保不同身份用户仅能访问其职责范围内的功能。

  • JWT令牌认证：登录成功后签发加密Token，用于后续接口的身份验证。
  • 密码安全存储：用户密码使用bcrypt算法哈希加密，杜绝明文风险。
  • 细粒度权限控制：每个API端点均设置角色访问策略，精确到具体操作级别。
  • 敏感操作防护：重要变更（如删除任务、重置密码）需二次确认，并记录审计日志。

四、项目文件结构说明

本项目采用清晰的模块化文件结构，便于开发、部署与维护。所有文件均按功能划分，确保前后端职责分明，配置与代码分离。以下是完整的项目目录树：

class-affairs-system/
├── index.html          # 主页面入口，包含系统核心导航与仪表盘
├── login.html          # 用户登录认证页面，支持四角色选择
├── tasks.html          # 班务收集管理前端页面
├── items.html          # 物资共享与借用管理页面
├── courses.html        # 课程情报查询与LLM问答界面
├── documents.html      # 文档资源共享与上传管理页面
├── admin.html          # 开发者专用后台管理面板
├── vercel.json         # Vercel平台部署配置文件
├── package.json        # Node.js项目依赖与脚本定义
├── .env.example        # 环境变量配置模板
├── README.md           # 本项目文档
└── api/                # Serverless Functions后端接口目录
    ├── users.js        # 用户管理API：认证、权限、CRUD操作
    ├── tasks.js        # 任务管理API：表单定义与提交处理
    ├── items.js        # 物资管理API：借用流程与状态变更
    ├── courses.js      # 课程管理API：情报发布与检索
    ├── documents.js    # 文档管理API：文件上传与版本控制
    ├── llm.js          # LLM问答API：智能检索与回答生成
    └── admin.js        # 后台管理API：系统配置与监控

核心文件说明

  • vercel.json：定义了路由重写规则（如/tasks/*指向tasks.html）、API函数映射（/api/(.*)）及静态资源缓存策略，是Vercel正确部署的关键。
  • package.json：声明了项目依赖（如Express、MongoDB驱动、OpenAI SDK等）和开发脚本（dev, build），确保环境一致性。
  • .env.example：提供了环境变量的标准模板，包括数据库连接、LLM API密钥、JWT签名密钥等敏感信息的占位符，部署时需复制为.env并填写实际值。
  • api/*.js：每个后端接口均为独立的Serverless Function，遵循RESTful规范，通过JWT中间件进行权限验证，实现高内聚、低耦合的服务架构。

该结构完全适配Vercel的自动化部署流程，开发者只需将此目录推送至GitHub仓库，即可触发一键上线。

本指南将引导您完成统双班务网页的完整部署流程，涵盖从GitHub仓库创建到Vercel上线的所有关键步骤。项目已实现全栈功能解耦，前端页面与后端API均采用标准格式输出，确保部署过程简洁高效。

请按照以下步骤将项目文件推送至GitHub，为Vercel部署做好准备。特别注意：工具生成的API文件（如api-users.js）需手动调整路径以符合项目结构要求。

步骤 操作说明 备注/提示

2. Vercel平台配置与部署

完成GitHub上传后，即可通过Vercel控制台导入项目并完成部署。整个过程无需本地安装额外依赖。

第一步：项目导入

  • 登录Vercel官网，推荐使用GitHub账号直接登录。
  • 点击“New Project”按钮，选择刚刚推送的 class-affairs-system 仓库。
  • Vercel会自动检测项目类型，确认框架预设为“Other”。

第二步：构建与输出配置

第三步：环境变量配置

在Vercel项目设置中，进入“Environment Variables”页面，添加以下关键变量：

变量名 示例值 说明

JWT_SECRET your-super-secret-jwt-key-change-in-production JWT令牌签名密钥，建议使用强随机字符串

第四步：启动部署

  • 点击“Deploy”按钮，Vercel将开始自动化部署流程。
  • 部署完成后，系统将分配一个默认域名（如 class-affairs-system.vercel.app），您可在“Settings”中绑定自定义域名。

至此，您的统双班务网页已成功上线，可通过浏览器访问主页面进行测试。

1. API接口文档

系统后端通过Vercel Serverless Functions暴露RESTful API接口，前端页面通过JavaScript调用这些接口实现数据交互。以下是各模块的核心API端点说明：

模块 端点 方法 功能描述 权限角色
用户管理 /api/users GET 获取用户列表（支持搜索与分页） admin, monitor
用户管理 /api/users/login POST 用户登录，返回JWT令牌 所有角色
任务管理 /api/tasks POST 创建新的班务收集任务 monitor
任务管理 /api/tasks/{task_id}/submissions POST 提交指定任务的表单数据 student
物资管理 /api/items/{item_id}/borrow POST 发起借用请求，更新物品状态为“已借出” student
物资管理 /api/items/{item_id}/return POST 发起归还请求，状态变更为“待确认归还” 借用人
课程管理 /api/courses/intel POST 发布课程相关情报（作业/考试等） student
课程管理 /api/courses/intel/{intel_id}/upvote POST 为情报条目点赞，提升其可信度 student
文档管理 /api/documents POST 上传新文档，支持PDF/Word/PPT等多种格式 student
文档管理 /api/documents/{doc_id}/download GET 触发文件下载，并记录下载次数 student
LLM问答 /api/llm/ask POST 提交自然语言问题，获取AI生成的回答 student
后台管理 /api/admin/config PUT 更新系统配置项，实时生效 admin

重要提示：所有需要身份验证的接口必须在请求头中包含 Authorization: Bearer <your-jwt-token>。

3. 使用指南和操作示例

以下为两个高频场景的操作流程示例：

场景一：班委发布一次信息收集任务

  1. 登录系统，选择“班委”角色。
  2. 进入“班务收集管理”页面，点击“创建新任务”。
  3. 填写任务标题（如“五一出行意向调查”）、描述，并设置截止时间。
  4. 在动态表单编辑器中添加字段：姓名（文本，必填）、是否参加（单选：是/否）、备注（多行文本）。
  5. 保存并发布任务，普通同学即可在首页看到待填写提醒。
  6. 截止后，班委可进入该任务详情页，点击“导出为Excel”获取所有提交数据。

场景二：同学查询本周作业安排

  1. 登录系统，选择“普通同学”角色。
  2. 进入“课程情报查询”页面，选择当前学期的“高等数学”课程。

5. 常见问题解答

Q1: 部署后前端页面正常，但无法登录或加载数据？
A: 请检查Vercel环境变量是否正确配置了MONGODB_URI和JWT_SECRET，并确认vercel.json中的API路由规则无误。

Q2: 如何修改文件上传大小限制？
A: 可在后台管理面板中调整“系统参数”，或直接修改.env文件中的MAX_UPLOAD_SIZE值（单位：MB），重启服务后生效。

Q3: LLM问答功能响应慢或失败？
A: 首先确认LLM_API_KEY有效且配额充足；其次检查网络是否能访问LLM服务商API；可在api/llm.js中调整请求超时设置。

Q4: 如何重置管理员密码？
A: 开发者可通过api-admin.js提供的POST /api/admin/users/{user_id}/reset-password接口重置任意用户密码，操作将被记录至审计日志。
